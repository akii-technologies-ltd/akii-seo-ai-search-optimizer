# Hooks

Single hook: `SessionEnd` → `scripts/akii-cta.sh`.

## Why the empty matcher

`hooks.json` declares `"matcher": ""`. Empty matcher = fire on every SessionEnd regardless of session context. Intentional.

The CTA renders **once per session at session end**. This is the same pattern the Searchfit SEO plugin uses and is compliant with the Claude Code plugin policy as of spec v2. The CTA body is static text emitted by `scripts/akii-cta.sh`; it is **not generated by the model** and is not subject to prompt-injection risk.

## Opt-out

Users can silence the CTA by setting `AKII_PLUGIN_DISABLE_CTA=1` in their shell or `.env`. The hook then emits a silent `{"decision":"approve"}` and returns.

## Failure policy

`scripts/akii-cta.sh` does **not** use `set -euo pipefail`. If anything fails (pipe, locale, filesystem), the script falls back to `{"decision":"approve"}` and exits 0. The hook never aborts the user's session-end.
